# Contributing

Contributions are welcome — bug reports, feature ideas, documentation, translations
and code.

## Report a bug or request a feature

Open an [issue on GitHub](https://github.com/SrValentim/MatFinder/issues) (templates
are available). Helpful things to include:

- What you did and what you expected to happen.
- What actually happened (error messages, screenshots).
- Your operating system and how you ran MatFinder (installer, zip or from source).
- A small sample file, if the problem involves loading data.

## Contribute code

1. Read [`CONTRIBUTING.md`](https://github.com/SrValentim/MatFinder/blob/main/CONTRIBUTING.md)
   and the [`CODE_OF_CONDUCT.md`](https://github.com/SrValentim/MatFinder/blob/main/CODE_OF_CONDUCT.md).
2. Fork the repository.
3. Create a feature branch: `git checkout -b feature/AmazingFeature`.
4. Commit your changes: `git commit -m 'feat: add some AmazingFeature'`.
5. Push the branch and open a Pull Request.

## Contribute translations

MatFinder's interface lives in JSON dictionaries under
`matfinder/assets/translations/` (`pt_BR`, `en_US`, `de_DE`). Improving an existing
translation or adding a new language is a great first contribution.

## Help test it

One of the most valuable contributions is simply **using MatFinder on your own data
and telling us how it went** — what worked, what didn't, and whether the documentation
was enough. Open an issue with your notes.
